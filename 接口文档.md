## 🚀 BCG 系统接口对接文档

### 1. 通用说明

- **基础路径 (Base URL):** `http://localhost:8080` (生产环境请根据实际配置修改)
- **认证方式:** 请在 Header 中携带 Token：`Authorization: Bearer {token}`
- **响应格式:** 统一返回 JSON。
  - `code`: 200 (成功), 500 (失败)
  - `msg`: 提示信息
  - `rows`: 列表数据 (仅限 list 接口)
  - `total`: 总条数 (仅限 list 接口)
  - `data`: 业务数据对象 (详情/新增/修改接口)

------

### 2. 设备管理模块 (`/device`)

| **功能**         | **方式** | **路径**         | **参数说明**                                                 |
| ---------------- | -------- | ---------------- | ------------------------------------------------------------ |

| `status`      | `Integer` | 处理状态 (0:处理中, 1:已完成) |

| `endTime`     | `String`  | 采集结束时间 (yyyy-MM-dd)     |

  - 若**找不到匹配信息**，后端将返回 `code: 404` 且不带 `rows`。

> ## 模块一：指标展示 (Metrics Display)
>
> 该模块包含 3 个接口，主要用于实现：**“选人 -> 出时间/心率列表 -> 选时间 -> 出仪表盘指标”** 的联动逻辑。
>
> ### 1. 获取检测人员名单
>
> 用于渲染左上角第一个“检测人姓名”下拉框。只会返回有成功检测数据的人员。
>
> - **接口路径:** `/bcg/datashow/subjects`
> - **请求方式:** `GET`
>
> **请求参数:**
>
> *(无参数)*
>
> **响应数据 (`data` 数组):**
>
> | **字段名**  | **类型** | **说明**                          |
> | ----------- | -------- | --------------------------------- |
> | subjectId   | Long     | 检测对象人员 ID，用于后续联动查询 |
> | subjectName | String   | 检测对象姓名                      |
>
> ------
>
> ### 2. 获取个人的历史检测记录
>
> 当用户在第一个下拉框选中人后，调用此接口获取该人员所有的检测时间及心率，用于渲染第二个下拉框。
>
> - **接口路径:** `/bcg/datashow/options/{subjectId}`
> - **请求方式:** `GET`
>
> **请求参数:**
>
> | **参数名** | **参数位置**    | **类型** | **必填** | **说明**                             |
> | ---------- | --------------- | -------- | -------- | ------------------------------------ |
> | subjectId  | Path (路径参数) | Long     | 是       | 检测对象的人员 ID (从上一个接口获取) |
>
> **响应数据 (`data` 数组):**
>
> | **字段名**     | **类型** | **说明**                                    |
> | -------------- | -------- | ------------------------------------------- |
> | id             | Long     | 该次检测记录的主键 ID，用于后续查指标和波形 |
> | collectionTime | String   | 数据采集时间 (格式: `yyyy-MM-dd HH:mm:ss`)  |
> | heartRate      | Double   | 该次检测的平均心率，可直接展示在下拉框里    |
>
> ------
>
> ### 3. 获取单次检测的核心健康指标
>
> 当用户在第二个下拉框选中具体的检测记录后，调用此接口获取 6 项核心健康指标，用于更新大屏仪表盘和卡片。
>
> - **接口路径:** `/bcg/datashow/metrics/{id}`
> - **请求方式:** `GET`
>
> **请求参数:**
>
> | **参数名** | **参数位置**    | **类型** | **必填** | **说明**                  |
> | ---------- | --------------- | -------- | -------- | ------------------------- |
> | id         | Path (路径参数) | Long     | 是       | 具体某次数据记录的主键 ID |
>
> **响应数据 (`data` 对象):**
>
> | **字段名**   | **类型** | **说明**                    |
> | ------------ | -------- | --------------------------- |
> | heartRate    | Double   | 平均心率 (bpm)              |
> | hrvSdnn      | Double   | 心率变异性 SDNN (ms)        |
> | respRate     | Double   | 呼吸频率 (次/分)            |
> | bodyMovement | Integer  | 体动次数                    |
> | sleepStatus  | String   | 睡眠状态评估 (如：平稳浅睡) |
> | healthScore  | Integer  | 综合健康评分 (0-100分)      |
>

> ## 模块二：波形预览 (Waveform Preview)
>
> 该模块包含 1 个接口，用于前端 ECharts 双图表的渲染。后端已对海量数据进行了**双路抽稀**和 **Redis 高速缓存**处理。
>
> ### 4. 获取双路波形图表数据
>
> 根据记录 ID 读取物理 CSV 文件，并返回适合 ECharts 渲染的抽稀坐标点阵。
>
> - **接口路径:** `/bcg/datashow/waveform/{id}`
> - **请求方式:** `GET`
>
> **请求参数:**
>
> | **参数名** | **参数位置**    | **类型** | **必填** | **说明**                  |
> | ---------- | --------------- | -------- | -------- | ------------------------- |
> | id         | Path (路径参数) | Long     | 是       | 具体某次数据记录的主键 ID |
>
> **响应数据 (`data` 对象):**
>
> | **字段名**    | **类型** | **说明**                                                     |
> | ------------- | -------- | ------------------------------------------------------------ |
> | originWave    | Array    | 原始波形数组：约 1500 个点，保留细节毛刺。适合使用 `type: 'line'` 渲染左侧图表。 |
> | processedWave | Array    | 预处理波形数组：约 200 个点，极为精简。适合使用 `type: 'line'` 并开启 `smooth: true` 渲染右侧图表。 |
>
> **波形数组内部结构说明 (originWave / processedWave 内部对象):**
>
> | **字段名** | **类型** | **说明**                                                     |
> | ---------- | -------- | ------------------------------------------------------------ |
> | x          | Integer  | 采样点序号 (从 0 开始递增)，前端直接作为 X 轴 (`type: 'value'`) |
> | y          | Double   | 对应的电压/幅值数据，前端作为 Y 轴                           |

| **参数名**  | **类型** | **必填** | **说明**           |
| ----------- | -------- | -------- | ------------------ |
| `subjectId` | Long     | **是**   | 被测人员/受试者 ID |

#### 1.2 查询参数 (Query Parameter)

| **参数名**  | **类型** | **必填** | **示例**     | **说明**                                                 |
| ----------- | -------- | -------- | ------------ | -------------------------------------------------------- |
| **`type`**  | Integer  | **是**   | `1`          | **核心字典字段：查询哪一项健康指标（详见下方字典说明）** |
| `beginTime` | String   | 否       | `2026-03-01` | 筛选开始时间 (不传则默认查全部历史记录)                  |
| `endTime`   | String   | 否       | `2026-03-31` | 筛选结束时间                                             |

### 2. 响应参数 (JSON)

#### 2.1 返回结构说明

| **参数名** | **类型**       | **说明**                                                 | **示例**                         |
| ---------- | -------------- | -------------------------------------------------------- | -------------------------------- |
| `code`     | Integer        | 状态码 (200为成功，404为时段内无数据)                    | `200`                            |
| `msg`      | String         | 提示信息                                                 | `"获取多维健康评估趋势成功"`     |
| `data`     | Object         | 统一扁平化图表数据载体                                   | `{...}`                          |
| ↳ `times`  | Array[String]  | **X 轴数据**：已按时间正序排列，精确到分钟               | `["04-01 10:00", "04-01 22:30"]` |
| ↳ `data`   | Array[Integer] | **Y 轴数据**：对应的状态值（**1 代表正常，0 代表异常**） | `[1, 1, 0, 1]`                   |

#### 2.2 响应成功示例

JSON

```
{
    "msg": "获取多维健康评估趋势成功",
    "code": 200,
    "data": {
        "times": [
            "04-01 00:00",
            "04-02 00:00",
            "04-03 00:00"
        ],
        "data": [
            1,
            1,
            0
        ]
    }
}
```

### 💡 前端组件复用与渲染建议 (最佳实践)

1. **图表组件极简封装**：

   由于四个维度的接口返回结构 `res.data.times` 和 `res.data.data` 是完全一致的。你只需要封装**一个** `<HealthTrendChart>` 组件，把 `type` 当作 props 传进去。

2. **极简赋值**：

   获取到接口数据后，直接无脑绑定：

   - `xAxis.data = res.data.times`
   - `series[0].data = res.data.data`

3. **视觉优化 (解决遮挡)**：

   由于值只有 `0` 和 `1`，极度推荐使用 **散点图 (`type: 'scatter'`)** 或者 **阶梯线图 (`step: 'start'`)**。如果是散点图，可以通过 ECharts 的 `itemStyle` 配置，让 `value === 1` 渲染为绿点，`value === 0` 渲染为红点，视觉效果最具医疗级专业感！
